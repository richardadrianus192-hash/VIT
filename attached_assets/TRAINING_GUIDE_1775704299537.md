# VIT Model Training — Step by Step Guide

## Before You Start (2 minutes)

You need:
- A Google account (for Colab)
- Your VIT project zip file OR a GitHub repo URL
- ~2 hours of free time (most of it unattended)

---

## Step 1 — Open Colab (2 minutes)

1. Go to https://colab.research.google.com
2. Click **File → Upload notebook**
3. Upload `VIT_Train_All_Models.ipynb`

OR paste this URL directly:
https://colab.research.google.com (then upload the .ipynb)

---

## Step 2 — Enable GPU (1 minute) ⚠️ CRITICAL

Without GPU, the LSTM, Transformer, GNN, and RL models will be
10x slower or may time out.

1. Click **Runtime** (top menu)
2. Click **Change runtime type**
3. Set **Hardware accelerator** to **T4 GPU**
4. Click **Save**

You should see "T4" in the top-right corner.

---

## Step 3 — Run Cells in Order

Run each cell top to bottom. Do NOT skip cells.

| Cell | What it does | Time |
|------|-------------|------|
| Step 1 | Checks GPU | 10s |
| Step 2 | Installs deps | 5–8 min |
| Step 3 | Upload project | 1–2 min |
| Step 4 | Download match data | 2–3 min |
| Step 5 | Train all 12 models | 60–120 min |
| Step 6 | Summary | 30s |
| Step 7 | Verify predictions | 2 min |
| Step 8 | Download / push | 1 min |

## Step 4 — Upload Your Project

## Step 6 — Download the Models

Run **Option A** in Step 8:
- A file called `vit_models.zip` will download automatically
- It contains all `.pkl` files + the training data

---

## Step 7 — Put Models Into Your Project

### Option A — Via GitHub (recommended)

```bash
# On your local machine after downloading vit_models.zip:
unzip vit_models.zip -d /tmp/vit_weights
cp /tmp/vit_weights/models/*.pkl YOUR_PROJECT/models/
git add models/*.pkl
git commit -m "Add trained model weights v1.0"
git push
```

Then in Replit Shell:
```bash
git pull
```
Restart the app. Check `/health` → `models_loaded` should be 10–12.

### Option B — Direct upload to Replit

1. In Replit file browser, navigate to `models/`
2. Click Upload
3. Select all `.pkl` files from the downloaded zip
4. Restart the app

## Troubleshooting

### "Module not found" error
The project path wasn't set correctly.
Fix: Re-run the "Setup Python path" cell, confirm `PROJECT_ROOT` points
to the folder containing `services/` and `app/`.

### LSTM/Transformer/GNN fails with CUDA error
Normal if GPU runs out of memory. The model falls back to CPU.
Re-run just that model's spec with a smaller batch size.

### PyMC (Bayesian) takes 30+ minutes
Normal. PyMC uses MCMC sampling which is slow but thorough.
If it times out, it will still save a partial model.

### Less than 500 matches downloaded
football-data.co.uk may be temporarily unavailable.
The notebook automatically tries extra seasons as fallback.
You can also manually download CSVs from football-data.co.uk
and upload them to Colab.

### Colab disconnects during training
Colab free tier disconnects after ~12 hours or if idle.
Re-run from Step 5. Already-saved `.pkl` files are preserved
in the session as long as you don't restart the runtime.

## Retrain Schedule

| Frequency | Benefit |
|-----------|---------|
| Weekly    | Captures recent form changes |
| Monthly   | Good balance of freshness vs stability |
| Each new season | Essential — team strengths shift significantly |

To retrain: re-run the notebook, it overwrites the old `.pkl` files.
